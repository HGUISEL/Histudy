3. Automatic Group Generating Algorithm

  • (located as Histudy/algorithm2.py )
  • The basic overall structure is as follows.

Step 1 : Generates small groups by grouping students who want to study together with other members

Step 2 : Each small group has its representation information which consist of (Number of Student, 1st preference, 2nd preference, 3rd preference and each small group is divided into 3 parts, Offline, Online and Off/Online.

Step 3 : We match with the Online and Online/Offline since it is the most usual case. And as the input passes through the “Group Matching Algorithm it generates outputs which are accumulated through the stack.

Step 4 : If the algorithm cannot generate groups with some of the small groups the leftovers are concat with the Offline members.

Step 5,6 : The concatenated input of Offline and leftovers pass through the Group Matching Algorithm and generates more groups and is saved in the stack

Group Matching Algorithm

At first we try to get several assumptions, that is, the maximum number of classes for each subject. This is an attempt to define the Maximum Class for the number of respects in the algorithm, but nevertheless, it seems impossible to implement realistically because it has a complexity of M^N (M is MinGroupNum - MaxGroupNum, N is the number of classes)

In addition, the number of cases in which the maximum profit of the first preference varies depending on the method of adding the elements of the second preference and the third preference occurs.

( which the complexity is M^N*O^P*Q^R, however, (P,R) will be a relatively smaller number than M)

Finally, the case of the maximum profit of the first preference is not greedy. (However, in most cases the algorithm is like to be greedy in most cases) Therefore, using Assumption is left uncharted for now. We made a new most likely assumption that making as many groups as possible would be the most efficient in most cases. In the current code, the assumption is set to Min = [ int(num / Min_number_of_member) for num in count].

The Group Matching Algorithm consists of 3 steps : first preference matching, second preference matching, Third preference matching.

  1. First preference matching

        The method of forming a group with the first preference consists of a total of three steps.

  1.  First, distribute the students as wide as possible by assigning them only to students with the same professor.
  2. By distributing it as widely as possible, some students may not be able to join the group due to the small number of students at the end. After distributing as widely we can put those students in the remaining space. However, some students might still have no remaining place
  3.  Even if it is not the same professor, put it in if there is an empty space.
  4.  When a group is formed with the first preference, there may be students who cannot join the group. The total number of remaining students in each subject must be less than the minimum group size.
  5.  This sequential matching method is advantageous for the student at the front of the list.

                    For fairness, the process of sorting the List is necessary before starting.

  1. Second preference matching

The method of forming a group with the second preference consists of a total of three steps.

  1. First, Explore whether there is a way to create a Minimum Group with the first and second preferences
  2.  At this time, sorting is performed so that the group with the smallest number of students is assigned first so that the rare group can be assigned first.
  3.  Distribute as many groups as possible.
  4.  Fill it again,
  5.  If there is space left to allocate in the existing group, allocate it there.

The reason why step 5 is not placed before step 1 is to ensure that as many students as possible can be assigned within the first and second preferences rather than being assigned to a class of the same professor.

  1. Third preference matching

The method of forming a group with the third preference also consists of a total of three steps. It is revised version of second preference matching that also consider the first and second preferences.
